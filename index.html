Template

Sample Bug Report Format

Severity: — Priority: —

Bug ID

BUG-XXXX-001

Reporter

Sadman Shakib Aungon

Environment

Browser / OS / Build version

Status

Open / In Progress / Fixed

Title

Short, clear description of the defect (e.g., "Login button unresponsive on mobile view")

Steps to Reproduce

  1. Go to [URL or page name]
  2. Perform [specific action]
  3. Enter [test data if needed]
  4. Click [button or element]
  5. Observe the result

Expected Result

What should happen according to requirements or common UX standards.

Actual Result

What actually happened — include error messages, screenshots, or console logs if available.

Severity & Priority Guide

Severity Meaning Priority Meaning
Critical System crash, data loss High Fix immediately
Major Feature broken, no workaround Medium Fix in current sprint
Minor Cosmetic or low impact Low Fix when possible
